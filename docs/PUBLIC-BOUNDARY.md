# Public / private boundary

LessonBuddy is developed as a private product. This repository is a deliberately reduced portfolio presentation.

## Included here

- a polished product case study;
- a small read-only FastAPI application;
- one typed public metadata response;
- responsive HTML/CSS/JavaScript;
- automated tests and CI;
- high-level interface mock-ups;
- an explicit record of what remains private.

## Kept private

- provider credentials and SDK integration;
- proprietary prompts and generation strategy;
- planning and generation bridges;
- commercial blueprint construction;
- detailed pedagogical validation and repair logic;
- private storage, history, and analytics;
- production print-fitting and document assembly;
- real teacher, school, or student data.

## Design rule

The public repository may explain product concepts and show visual mock-ups, but it must not reproduce the working generation system.

The public API is read-only. The browser cannot submit lesson briefs or request generation. Automated tests also check that known private-product markers and write routes are absent.
